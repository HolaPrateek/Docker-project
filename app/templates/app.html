Your Own WebServer inside a docker container

In order to understand, how to configure your own web server inside a docker container first of all you should know:

What is a Web Server?

A web server includes several parts that control how web users access hosted files, at minimum an HTTP server. An HTTP server is a piece of software that understands URLs (web addresses) and HTTP (the protocol your browser uses to view webpages). It can be accessed through the domain names (like mozilla.org) of websites it stores, and delivers their content to the end-user's device.

At the most basic level, whenever a browser needs a file which is hosted on a web server, the browser requests the file via HTTP. When the request reaches the correct web server (hardware), the HTTP server (software) accepts request, finds the requested document (if it doesn't then a 404 response is returned), and sends it back to the browser, also through HTTP.

Requirements to setup you web server:

  • RedHat Linux RHEL8
  • Docker
  • Docker Image
  • Apache web server

Installation:
Make sure you have the latest version of Docker installed on your machine.

If Not than run the following commands:-

For Docker installation on linux[Redhat]
  • dnf list docker-ce

For Docker Installation on RedHat/Centos:
  • Configure yum by adding docker.repo and dvd.repo inside the /etc/yum.repos.d for local installation using
    • https://download.docker.com/linux/centos/docker-ce.repo
  • Now, run command yum install docker-ce --nobest
Start Docker: Start your docker services by using command
  • systemctl start services
Creating container inside Docker:
 To create a container inside a docker you should have an os, which is called docker image. You can either download it from hub.docker.com
Or you can use the following command
  • docker pull image_name
    Here, image name refers to on which you want to make your server e.g. ubuntu, centos etc. Now to deploy container inside docker use command
    • docker container run -t -i --name ____ docker_img_name:version
    her you can give name to the server. Ex- docker container run -t -i --name web centos:8
    Configure your Apche Server:
    • First of all, install apache server inside you container using the command:
      dnf install httpd
    • Second, Start your apache services using command:
      systemctl start httpd
    • Third, turn off firewall services so that your clients can access your server by using command:
      systemctl stop firewalld.services
    • Third, configure your selinux settings using command:
      setenforce 0
    • Fourth, restart your apache services using command:
      systemctl restarthttpd
      • Your server is configured now. Your clients can access you site now. Now save your container by copying it so that your server don't get eliminate.Now exit the container by using "exit" command. Now to copy your container having your web server follow the given command:
      docker commit container_name your_new_container_name
